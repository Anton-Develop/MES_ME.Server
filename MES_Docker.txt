Хорошо, Антон. Теперь, когда у нас есть общее понимание структуры (бэкенд на .NET Core, фронтенд на React, оба в Docker-контейнерах, фронтенд использует Nginx для проксирования API к бэкенду), давай разберем, **какие команды выполнять при изменении каждой части**.

### Сценарий 1: Вы внесли изменения в **бэкенд** (код .NET Core, контроллеры, модели, логика, `Program.cs`, `appsettings.*.json` и т.д.).

Когда вы изменяете *код* или *конфигурацию* бэкенда, вам нужно **пересобрать Docker-образ** для бэкенда и **перезапустить соответствующий контейнер**. Предположим, вы находитесь в директории `/opt/mes_app/backend/`, где расположен `docker-compose.mes.yml`.

**Команды:**

1.  **Остановить текущий контейнер бэкенда (и, возможно, связанные, если используете `down`):**
    ```bash
    sudo docker compose -f docker-compose.mes.yml down
    ```
    *(Команда `down` удаляет контейнеры, но оставляет образы и сеть.)*

2.  **Пересобрать образ и запустить контейнер:**
    ```bash
    sudo docker compose -f docker-compose.mes.yml up --build -d
    ```
    *   `--build`: Заставляет Docker пересобрать образ, даже если он уже существует. Это важно, чтобы изменения в коде или `Dockerfile` попали в новый образ.
    *   `-d`: Запускает контейнер в фоновом режиме.

**Альтернатива (если сеть `mes_app_network` уже создана и вы не хотите её останавливать):**

*   **Принудительно пересоздать и перезапустить ТОЛЬКО сервис `mesmeserver` (бэкенд):**
    ```bash
    sudo docker compose -f docker-compose.mes.yml up --build -d --force-recreate mesmeserver
    ```
    *(Замените `mesmeserver` на имя сервиса, как оно определено в `docker-compose.mes.yml`)*

### Сценарий 2: Вы внесли изменения в **фронтенд** (код React, компоненты, стили, логика, `nginx.conf` и т.д.).

Когда вы изменяете *код* фронтенда или *конфигурацию Nginx*, вам нужно **пересобрать Docker-образ** для фронтенда и **перезапустить соответствующий контейнер**. Предположим, вы находитесь в директории `/opt/mes_app/frontend/`, где находятся `Dockerfile`, `nginx.conf` и `docker-compose.yml` (или как он у вас называется).

**Команды:**

1.  **Остановить текущий контейнер фронтенда:**
    ```bash
    sudo docker compose -f docker-compose.yml down # Используйте имя вашего compose файла
    ```

2.  **Пересобрать образ и запустить контейнер:**
    ```bash
    sudo docker compose -f docker-compose.yml up --build -d # Используйте имя вашего compose файла
    ```
    *   `--build`: Пересобирает образ с учетом изменений в `Dockerfile` и `nginx.conf`.
    *   `-d`: Запускает контейнер в фоновом режиме.

**Альтернатива (если сеть `mes_app_network` уже создана и используется другими сервисами):**

*   **Принудительно пересоздать и перезапустить ТОЛЬКО сервис фронтенда (например, `frontend-react`):**
    ```bash
    sudo docker compose -f docker-compose.yml up --build -d --force-recreate frontend-react # Замените на имя вашего сервиса фронтенда
    ```

### Важные моменты:

*   **`--build`:** Эта опция *критически важна* всякий раз, когда вы изменяете файлы, которые учитываются при сборке Docker-образа (`Dockerfile`, `nginx.conf`, исходный код .NET/React).
*   **`--force-recreate`:** Полезно, если вы хотите *гарантировать* пересоздание контейнера, даже если образ не изменился. Обычно `up --build` этого делает, но `--force-recreate` исключает все сомнения.
*   **Директории:** Всегда выполняйте команды `docker compose` из той директории, где находится *соответствующий* файл `docker-compose.yml`.
*   **Папка `build/`:** Если вы используете предложенный метод с локальной сборкой `npm run build` и копированием папки `build`, то при *изменении* кода React, вы сначала должны снова выполнить `npm run build` *локально*, скопировать новую папку `build` на сервер в `/opt/mes_app/frontend/`, а *затем* выполнить команды `docker compose` для пересборки образа и перезапуска контейнера.
*   **CORS:** Если вы вносите изменения в настройки CORS в бэкенде, убедитесь, что они отражают *новый* Origin фронтенда (например, если IP-адрес сервера изменится или вы решите использовать домен).
*   **Подключение к БД:** Изменения в строке подключения к БД (`ConnectionStrings__DefaultConnection`) обычно передаются через переменные окружения в `docker-compose.mes.yml`. Изменение этой переменной в compose файле также требует перезапуска бэкенд-контейнера (`sudo docker compose -f docker-compose.mes.yml up --build -d` или `down` и `up`).

Таким образом, для каждого изменения нужно понимать, к какой части (бэкенд или фронтенд) оно относится, и выполнять соответствующие команды в нужной директории.
